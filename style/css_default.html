CSS란?

Cascading Style Sheets 의 약자로
css,스타일,스타일시트 라고도 부른다.

- HTML문서를 꾸며주는 역할을 한다.
- 화면의 레이아웃을 잡는 역할
- 레이아웃을 잡을때 div나 시맨틱 태그를 기분으로 해서 사용
- HTML5에서는 시맨틱 태그를 사용 해야함.(단순히 나눠주는 div로 레이아웃을 하지 않음.)

ex) < div class="header" >

위 예시 처럼 작성되어진 경우가 종종 보이는데.. 요건 HTML4.0입니다.
HTML5에서는 header라는 태그가 존재하기 때문에 위처럼 작성하는게 아닌 header태그를 사용해서 마크업 하면 됨.

CSS(Cascading Style Sheets)

사용방법


선택자 {
		속성명1 : 값1;
		속성명2 : 값2;
		속성명n : 값n;
		}
	h1 {
		color;red;
		background:yellow
		}
	
선택자(selector) : 스타일시트가 적용될 범위를 지정하는 키워드
	태그, 클래스명, 아이디명, 태그속성명 등등....
	
속성(property) : 스타일시트의 속성으로 대체적으로 두 단어 이상 연결 시 하이픈(-)을 이용한다.
	
	cf) html : bgcolor
		css : background-color
		js : bgcolor

값, 속성값(values) : 속성에 해당하는 값으로 여러개일 경우가 많음.
	ex) background-color 의 속성값으로 .. red, green,blue... 등등 온갖 생상 명칭이 들어갈 수 있음.

웹문서에 CSS 적용하는 기본적인 방법 + 우선순위

3순위) 외부파일 링크 (외부 스타일 시트)

: ~.css파일 생성 후 html 파일 내에서 링크 태그로 연결 해서 사용하는 방법

< link href="css파일의 경로" rel="stylesheet" type="text/css" />
2순위) 웹문서 head 태그 안에 기술 (내부 스타일 시트)
<style>
	...................
<style>


★1순위) 태그 안 사용 (인라인 스타일 시트)

< 태그명 style="속성명1:값1; 속성명n:값n;" >

인라인 스타일 시트는 무조건 우선순위가 가장 높음.

외부,내부 스타일 시트의 경우 작성된 텍스트 중
가장 아래에 기술하는 것이 우선순위가 더 높음.

★클래스와 아이디★

아이디 (id) : id="아이디명"

하나의 화면(웹문서)에서 중복되는 값이 존재하면 안되는 선택자
- 중복된다고 해서 브라우저상에서는 오류를 표기하거나 그러진 않음.
마크업 유효성 검사를 한다거나 해당 아이디로 동작되어지는 js가 들어간다면 문제가 발생함
(동적효과를 넣을때 내가 예상한 결과와는 다른 결과가 나올 수도 있다.)
- a태그의 href 속성값으로 #아이디명을 쓰게되면 '내부링크(북마크)'가 되어 해당 아이디가 있는 위치로 스크롤이 이동 되어짐.


클래스(class) : class="클래스명"

- 클래스명의 명명규칙은 아이디와 같으나, 문서 내에서 중복되는 값이 존재 해도 됨.
- 다른 요소(태그)라도 같은 이름의 클래스로 묶어서 사용 가능.
